## 2. 변경 / 신규 기능 요구사항 (Functional Requirements)

### 2.1 변경: FR-1.1 게임방 생성 (Updated, supersedes v1.1 FR-1.1)

- 서버 시작 시 게임방은 **즉시 생성되지 않는다**. 서버는 "호스트 진입 대기" 상태로 시작한다.
- 첫 번째 사용자가 `/public` 에 접속하면 그 브라우저 세션에 **호스트 권한**이 부여된다 (CR1-Q10=C). 호스트 권한 부여는 자동, 사용자 인증 없음.
- 호스트 화면에는 **"방 개설"** 버튼과 **게임 설정 입력 폼**(FR-11 참조)이 노출된다.
- 호스트가 게임 설정 입력 후 **"방 개설"** 을 명시적으로 클릭해야 게임방이 생성되어 플레이어 입장이 허용된다 (CR1-Q1=A).
- 방이 개설된 시점부터 `/play` 입장이 가능하다 (FR-1.2 참조).
- 게임방은 **서버 인스턴스당 동시 1개만 운영**한다 (CR1-Q2=B, FR-10 참조).

### 2.2 변경: FR-1.2 플레이어 입장 (Updated, supersedes v1.1 FR-1.2)

- **방이 아직 개설되지 않은 상태**에서 `/play` 에 접속한 사용자는 닉네임 입력 화면 대신 **"방이 아직 없습니다. 호스트가 방을 개설할 때까지 기다려 주세요."** 안내 화면을 본다 (CR1-Q1=A 보완). 자동 새로고침 없이도 방이 개설되면 안내가 자동 갱신된다(WebSocket 푸시).
- 방 개설 후의 입장 규칙은 v1.1 FR-1.2 와 동일 (닉네임 입력, 중복 거부, 대기실 진입).
- 입장 시 **호스트는 플레이어로 자동 참가하지 않는다** (FR-9 참조).

### 2.3 변경: FR-1.4 게임 시작 (Updated, supersedes v1.1 FR-1.4)

- 게임 시작 권한은 **호스트(첫 `/public` 접속자) 만**이 가진다.
- 호스트는 최소 인원(FR-1.3 의 6명 하한, FR-11 의 호스트 지정 최대치 이하) 도달 시 **"게임 시작"** 버튼을 활성 상태로 클릭할 수 있다.
- 게임 시작 시 시스템이 FR-11 의 게임 설정을 토대로 역할과 키워드를 자동 배정한다.

### 2.4 변경: FR-2.2 역할 배분 규칙 (Updated, supersedes v1.1 FR-2.2)

- v1.1 의 인원수별 자동 배분 테이블은 **권장값(recommended preset)** 으로 격하된다.
- 실제 배분은 호스트의 **게임 설정 (FR-11)** 입력값을 사용한다.
- 의사·경찰은 각 1명으로 고정 (변경 없음). 마피아 수 = 호스트 입력값. 시민 수 = 참여 인원 − 마피아 수 − 의사 1 − 경찰 1.
- 마피아 수 입력값에 대한 검증/경고 정책은 FR-11.3 참조.

### 2.5 변경: FR-4.1 첫째날 — 자기소개 단계 (Updated, supersedes v1.1 FR-4.1)

- 첫째날 낮의 자기소개 단계 진입은 **시스템이 자동**으로 수행한다 (호스트 클릭 없음 — CHG-9).
- 자기소개 발언 순서는 시스템이 **입장 순 자동 라운드 로빈**으로 결정한다 (CR1-Q8=A). 호스트는 발언자 선정에 개입하지 않는다.
- 발언 중인 플레이어의 `/play` 화면에는 **"내 자기소개 종료"** 버튼이 활성화된다 (CR1-Q7=A). 본인이 클릭하면 다음 순번으로 자동 진행된다.
- 모든 살아있는 플레이어의 자기소개가 끝나면 시스템은 **자동으로 밤 단계로 전환**한다 (호스트 클릭 없음).
- (참고) 발언자 본인이 자리에 없거나 종료 클릭을 누르지 못해 진행이 막히는 케이스의 회복 정책은 본 반복 Out of Scope. 다음 반복 또는 강제 종료(CHG-9 ㉧)로 처리.

### 2.6 신규: FR-9 호스트 역할 분리

#### FR-9.1 호스트 정의
- **호스트(Host)** 는 게임의 GM(Game Master) 역할만 수행한다. 게임 자체에는 플레이어로 참여하지 않는다 (CR1-Q3=C 변형).
- 호스트 권한은 **첫 `/public` 접속 브라우저 세션** 에만 부여된다 (FR-10).

#### FR-9.2 호스트 GM 책임
호스트가 GM으로서 수행하는 행동은 **세 가지로 한정된다** (CR1-Q9=B):
1. 방 개설 / 게임 설정 입력 (FR-1.1, FR-11)
2. 게임 시작 (FR-1.4)
3. 게임 강제 종료 / 무효화 (FR-9.4)

그 외 모든 게임 진행 단계(자기소개·밤·낮·투표·결과)는 시스템이 자동으로 전환한다 (CHG-9).

#### FR-9.3 호스트의 게임 참여 (보조 채널)
- 호스트가 본인도 플레이어로 게임에 참여하고자 할 경우, **별도 기기/브라우저로 `/play` 에 접속**하여 일반 플레이어로 입장한다 (CR1-Q3=C). 이때 시스템은 그 `/play` 세션을 호스트 권한 세션과 구분하지 않으며, 일반 플레이어와 동일하게 취급한다.
- 호스트가 별도 기기 없이 본인의 호스트 PC 한 대에서 동시에 게임에 참여하는 기능은 **본 반복에서 제공하지 않는다** (Out of Scope).

#### FR-9.4 게임 강제 종료
- 호스트는 게임 진행 중 언제든 `/public` 화면에서 **"강제 종료"** 버튼으로 게임을 종결할 수 있다.
- 강제 종료된 게임은 결과를 정상 종료와 구분하여 기록한다 (FR-6 결과 저장 정책에 'forced_terminated' 상태 추가). 강제 종료 시 키워드/역할 공개 여부 등 세부 정책은 본 반복 Out of Scope (다음 반복에서 결정).

### 2.7 신규: FR-10 단일 방 운영 / 호스트 인증

#### FR-10.1 단일 방 정책
- 서버 인스턴스당 동시에 운영 가능한 게임방은 **1개**다 (CR1-Q2=B).
- 게임이 정상/강제 종료되면 새 방을 동일 서버 인스턴스에서 다시 개설할 수 있다 (방 라이프사이클 종료 후 다시 호스트 진입 대기 상태).

#### FR-10.2 호스트 인증 — 첫 접속자
- **첫 번째 `/public` 접속자만 호스트 권한을 가진다** (CR1-Q10=C). 별도 비밀번호·계정 인증 없음.
- 두 번째 이후의 `/public` 접속은 **"이미 호스트가 방을 운영 중입니다"** 안내 화면을 보여주며 호스트 권한을 부여하지 않고 입장도 차단한다 (CR1-Q2=B). read-only 관전 페이지 제공은 본 반복 Out of Scope.
- 호스트 브라우저 세션이 종료(탭 닫기·네트워크 끊김 등)되면 호스트 권한이 회수된다. 회수 후의 동작(다음 `/public` 접속자가 호스트가 되는지, 게임이 멈추는지 등)은 본 반복 Out of Scope (현재 게임이 진행 중이면 강제 종료에 준해 처리하는 것을 다음 반복에서 정의).

### 2.8 신규: FR-11 게임 설정

#### FR-11.1 설정 입력 시점
- 게임 설정은 호스트가 `/public` 의 **방 개설 화면에서 1회 입력**한다.
- 일단 **"방 개설"** 클릭 후의 설정 변경은 **본 반복 Out of Scope** (방 종료 후 새 방 개설 시 다시 입력).

#### FR-11.2 설정 항목
| 항목 | 입력 방식 | 허용 범위 | 기본값 |
|---|---|---|---|
| 최대 참여 인원 | 정수 선택 | 6 ~ 12 (CR1-Q4=A) | 6 |
| 마피아 수 | 정수 선택 | 1 ~ (참여 인원 − 의사 1 − 경찰 1 − 시민 1) (CR1-Q5=C 보완) | 인원수별 권장값 (v1.1 FR-2.2 표 사용) |

#### FR-11.3 마피아 수 검증/경고 정책 (CR1-Q5=C + CR1-Q6=C 통합)
- UI는 인원수에 따른 **권장 마피아 수**를 기본값으로 표시한다.
- 호스트는 권장값 ±1 범위는 클릭 한 번(증감 버튼)으로 손쉽게 조정할 수 있다.
- ±1 범위 외의 값도 직접 입력으로 지정 가능하나, **권장 범위(±1) 를 벗어난 경우 "권장하지 않는 설정입니다" 경고를 표시**한다.
- 경고 표시 후에도 호스트는 **그대로 게임을 시작할 수 있다** (강행 허용).
- 마피아 수 입력값이 FR-11.2 의 허용 범위(시민 ≥ 1, 의사·경찰 1명 확보 가능)를 벗어나면 **에러로 차단**한다.

### 2.9 신규: FR-12 자기소개 턴 흐름 (FR-4.1 변경의 신규 ID 부여)

#### FR-12.1 발언자 자동 진행
- 시스템은 자기소개 단계 진입 시 입장 순으로 첫 발언자를 결정한다.
- 현재 발언자의 `/play` 화면에 **"내 자기소개 종료"** 버튼이 활성화된다.
- 다른 플레이어의 `/play` 화면 및 `/public` 화면에는 현재 발언자가 누구인지 표시된다.

#### FR-12.2 발언자 종료 → 다음 진행
- 발언자가 본인 기기의 **"내 자기소개 종료"** 버튼을 클릭하면 시스템은 다음 발언자로 자동 전환한다.
- 마지막 발언자가 종료를 클릭하면 시스템은 자동으로 첫 번째 밤 단계로 전환한다 (FR-4.2).

#### FR-12.3 호스트 개입 없음
- 자기소개 단계 진행에서 호스트의 클릭은 필요하지 않다. 호스트 화면(`/public`)은 진행 상태(현재 발언자, 남은 인원)를 모니터링만 한다.
- 발언자가 종료 클릭을 누르지 못해 정체되는 경우 호스트는 게임 강제 종료(FR-9.4)로만 개입할 수 있다 (다른 우회 진행 수단은 본 반복 Out of Scope).

| NFR | 영향 | 변경 내용 |
|---|---|---|
| **NFR-2 신뢰성** | 직접 영향 | 단일 방 강제(FR-10.1)는 동시성 락(v1.1 NFR-2 "단일 GM 보장")을 호스트 권한 부여 단계로 확장. 두 번째 `/public` 접속자에게 일관된 거부 화면 보장. |
| **NFR-3 사용성** | 직접 영향 | 호스트 GM 화면 신규 UI 톤은 **사회자/진행자 톤**으로 통일 (CR1-Q11=B). 예: "방을 개설합니다", "참가자를 받습니다", "설정을 마쳤다면 게임을 시작합니다". 플레이어 화면의 "내 자기소개 종료" 버튼은 발언 중인 플레이어에게만 명확히 활성화 표시. |
| **NFR-4 보안** | 약영향 | 첫 접속자 호스트 모델(FR-10.2)은 인증 없는 신뢰 모델 유지(사내 LAN 가정). Security Baseline 확장 비활성화 결정과 일관. |
| **NFR-6 유지보수성** | 약영향 | 자기소개 자동 진행 도입으로 기존 "다음 발언자" 호스트 클릭 로직은 제거되거나 자동 트리거로 대체. |

본 patch에서 명시되지 않은 NFR(성능, 호환성, 운영 단순성, 데이터 무결성)은 v1.1 그대로 유효.

## 4. 사용자 시나리오 갱신 (변경된 골든패스만)

### 시나리오 1 (갱신): 호스트 시작 — 사회자 단독 GM 모드

1. 호스트가 자기 노트북에서 Go 바이너리 실행 → 콘솔에 `http://192.168.0.10:8080` 표시.
2. 호스트는 노트북을 회의실 모니터+스피커에 연결, 브라우저로 `/public` 접속 → **호스트 권한 부여, "방 개설" 화면 표시**.
3. 호스트가 게임 설정 입력: 최대 인원 8, 마피아 수 권장값 2 그대로 → **"방 개설"** 클릭. 공용 화면이 "참가자를 받습니다"(사회자 톤) 표시로 전환.
4. 다른 참가자 7명이 자기 휴대폰/노트북으로 `/play` 접속 → 닉네임 입력 후 입장. (호스트는 본인 게임 참여 안 함)
5. 호스트가 8명(최대 인원) 충족 시점에 **"게임 시작"** 클릭 → 시스템이 마피아 2 / 의사 1 / 경찰 1 / 시민 4 자동 배정 + 키워드 부여.
6. 공용 화면 자동으로 "1일째 — 자기소개" 표시 + TTS "첫째날 낮입니다. 차례대로 자기소개를 해주세요." → 첫 발언자(입장 순 1번)의 `/play` 에 "내 자기소개 종료" 활성화.
7. 발언자가 본인 기기 종료 클릭 → 자동으로 다음 발언자로 전환. 모두 끝나면 자동 밤 단계 전환 (호스트 클릭 없음).
8. 이후 밤·낮·투표·결과 모든 단계는 시스템 자동 전환. 호스트는 화면 모니터링만 수행.

### 시나리오 2 (갱신): 두 번째 호스트 접속 거부

- 첫 호스트가 이미 `/public` 에 접속한 상태에서 다른 사용자가 `/public` 접속 → "이미 호스트가 방을 운영 중입니다" 안내, 호스트 권한 미부여, 게임 화면 진입 불가.

### 시나리오 3 (신규): 호스트가 별도 기기로 게임 참여

- 호스트가 회의실 큰 모니터에서 `/public` 으로 GM 진행 중 + 본인 휴대폰으로 `/play` 접속하여 닉네임 입력 후 일반 플레이어로 참가. 시스템은 휴대폰 세션을 일반 플레이어로 처리.
- 게임 진행 중 호스트는 휴대폰으로 자기소개·투표 등 일반 플레이어 액션을 수행하면서, 큰 모니터(`/public`)로는 GM 모니터링과 필요 시 강제 종료를 수행.

### 시나리오 4 (갱신): 호스트 PC 재시작

- 호스트 PC가 도중에 재부팅된 경우, 서버를 다시 띄우고 호스트가 다시 `/public` 접속하면 호스트 권한이 첫 접속 규칙에 따라 재부여된다. 게임 상태 복원은 v1.1 시나리오 3과 동일 (last persisted snapshot).

### 시나리오 5 (신규): 자기소개 강제 종료 (예외)

- 어떤 발언자가 자리를 비워 "내 자기소개 종료" 클릭이 되지 않을 경우, 본 반복에서는 자동 회복 수단을 제공하지 않는다. 호스트는 **게임 강제 종료(FR-9.4)** 로만 개입할 수 있다.

기타 시나리오(승부 종료·결과 조회·음성 OFF 토글 등)는 v1.1 그대로 유효.
